Plan Maestro Interactivo

Una guía paso a paso para la implementación de IBM Test Accelerator for Z.

1.1 Visión Estratégica y Propuesta de Valor

Abordar los desafíos de las pruebas en z/OS mediante la aceleración del desarrollo (Shift-Left), automatización integral y la eliminación de dependencias con virtualización de subsistemas.

1.2 Arquitectura y Componentes Centrales

  • Dynamic Test Engine: Motor de grabación y reproducción en z/OS.
  • Early Development Testing (EDT): Pruebas unitarias para desarrolladores en IDz.
  • Integration Test Builder (ITB): Pruebas de integración en VS Code con Galasa.
  • On-demand Dev/Test (ODTz): Entornos z/OS virtualizados bajo demanda.

1.3 Prerrequisitos y Roles

Coordinación precisa entre roles: Líder de Proyecto, Programador de Sistemas, Administrador de Seguridad (RACF), Administrador de Red, Administrador CICS/IMS y equipos de Desarrollo.

2.1 Instalación y Configuración Central

Instalación SMP/E, personalización de PARMLIB (IFAPRDxx, LPALSTxx, PROGxx), configuración de PROCLIB para el Debug Profile Service (EQAPROF) e integración con CICS e IMS.

2.2 Configuración de Seguridad y Red

Creación de perfiles RACF para la tarea iniciada (EQAPROF), concesión de privilegios de z/OS UNIX y configuración de comunicación cifrada con certificados digitales y llaveros (key rings).

Despliegue de herramientas cliente como dos sub-proyectos distintos: uno para desarrolladores tradicionales (EDT en IDz) y otro para testers/desarrolladores modernos (ITB en VS Code).

Guía práctica para ejecutar pruebas desde EDT e ITB, interpretar resultados y utilizar la lista de verificación para la resolución de errores comunes (conectividad, logs, permisos, etc.).

Maximizar el valor de la inversión a largo plazo mediante la integración en pipelines de CI/CD, la gestión de código fuente en Git y el establecimiento de un Centro de Excelencia (CoE).
